## Key Decisions Documented

| Area | Decision | Rationale |
|------|----------|-----------|
| **Image Storage** | Browser compression → Base64 in IndexedDB → file on server | Simple, no external services, mobile-friendly |
| **Sync Strategy** | Event queue + LWW conflict resolution | Deterministic, supports single-device use case |
| **Service Worker** | SvelteKit native `sw.ts` + Workbox | Built-in, handles cache invalidation |
| **Authentication** | Session-based cookies | Simple, CSRF-protected, SvelteKit integrated |
| **Offline Storage** | IndexedDB (main) + localStorage (metadata) | Sufficient quota (~50MB), supports querying |
| **Database ORM** | Drizzle + PostgreSQL | Type-safe, explicit schema, good DX |
| **Testing** | Vitest (unit) + Playwright (E2E) | SvelteKit standard, good coverage |
| **Project Structure** | SvelteKit monorepo (single src/) | Simple, focused, maintains modularity |
